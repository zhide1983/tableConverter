🧪 TableClip 合并单元格测试

测试说明： 这个页面用于测试和验证合并单元格的修复效果。 下面是一个包含合并单元格的测试表格，以及期望的输出格式。

📊 测试表格 (A-C列，1-4行，A2~A4合并)

列A 列B 列C
合并内容 B2 C2
B3 C3
B4 C4

说明： A2单元格向下合并到A4，包含文本"合并内容"

✅ 期望的 Markdown 输出

合并单元格内容应该显示在左上角第一个位置（A2），其他位置（A3、A4）留空：

| 列A | 列B | 列C | | --- | --- | --- | | 合并内容 (1×3) | B2 | C2 | | | B3 | C3 | | | B4 | C4 |

✅ 期望的 HTML 输出

HTML 格式应该正确使用 rowspan 属性：

<table> <thead> <tr> <th>列A</th> <th>列B</th> <th>列C</th> </tr> </thead> <tbody> <tr> <td rowspan="3">合并内容</td> <td>B2</td> <td>C2</td> </tr> <tr> <td>B3</td> <td>C3</td> </tr> <tr> <td>B4</td> <td>C4</td> </tr> </tbody> </table>

✅ 期望的 Textile 输出

Textile 格式使用 /n. 语法表示行合并：

|_. 列A|_. 列B|_. 列C| |/3. 合并内容|B2|C2| |B3|C3| |B4|C4|

✅ 期望的 RST 输出

RST 格式应该生成正确的边框和对齐：

+=================+=====+=====+ | 列A | 列B | 列C | +=================+=====+=====+ | 合并内容 [1×3] | B2 | C2 | | +-----+-----+ | | B3 | C3 | + +-----+-----+ | | B4 | C4 | +=================+=====+=====+

🧪 如何测试

  1. 复制上面的测试表格（选择包括表头的整个表格）
  2. 打开 TableClip 主页面
  3. 点击"粘贴表格数据"按钮
  4. 切换不同的输出格式，检查是否与期望输出一致
  5. 特别关注：
    • Markdown: 合并区域的内容是否只在第一个位置显示，其他位置是否为空
    • RST: 是否还会报错，格式是否正确
    • HTML & Textile: 是否保持原有的正确功能
